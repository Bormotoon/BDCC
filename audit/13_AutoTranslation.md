# Auto Translation System — Analysis

> `Util/AutoTranslation/` — 10 файлов, ~1,020 строк

## 1. Обзор

Мультибэкенд система перевода с цепочкой фоллбэков. Поддерживает 4 бэкенда переводчиков.

## 2. Архитектура

```
AutoTranslation.gd (246 строк) — координатор
  ├── GoogleTranslator
  ├── GoogleBatchTranslator
  ├── PapagoTranslate (Naver)
  ├── MicrosoftTranslator (Bing)
  └── DeepLTranslator
```

### 2.1 Цепочка фоллбэков

```gdscript
# Порядок переводчиков настраивается пользователем
# Если один падает — используется следующий
```

### 2.2 Разбивка текста

- Текст разбивается по новым строкам
- Максимальный размер чанка: **4000 символов**
- Задержка между чанками: **2 секунды**
- Для batch переводчиков — объединение запросов

### 2.3 Пользовательская настройка

- Порядок переводчиков настраивается через UI
- Можно добавить/удалить/переставить переводчики
- Up/Down reorder в интерфейсе

## 3. Бэкенды

| Бэкенд | API | Особенности |
|--------|-----|-------------|
| Google | Free/DeepL | Простой, по умолчанию |
| GoogleBatch | Free | Пакетная отправка |
| Papago | Naver | Корейский API |
| Microsoft/Bing | Azure | Требует ключ |
| DeepL | DeepL API | Высокое качество |

## 4. Сильные стороны

1. **Цепочка фоллбэков** — отказоустойчивость
2. **Настраиваемый порядок** — пользователь контролирует
3. **Chunking** — обработка длинных текстов
4. **Multiple backends** — 4 опции

## 5. Слабые стороны

1. **2-секундная задержка** — ощутимо для больших текстов
2. **Нет кэширования** — повторный перевод одного текста
3. **Нет retry** — падение = переход к следующему
4. **Нет rate limiting** — можно превысить лимиты API
5. **Hardcoded chunk size** — 4000 символов без настройки

## 6. Рекомендации

| Приоритет | Действие |
|---|---|
| **Средний** | Добавить кэширование переведённых текстов |
| **Средний** | Добавить retry с exponential backoff |
| **Низкий** | Сделать chunk size настраиваемым |
| **Низкий** | Добавить rate limiting per backend |
